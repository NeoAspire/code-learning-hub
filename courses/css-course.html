What is CSS?

CSS (Cascading Style Sheets) is a language used to style and design web pages. It controls how HTML elements look, including:

  • Colors
  • Fonts and text size
  • Layouts and spacing
  • Backgrounds and images
  • Animations and effects

In short: HTML is the structure of a webpage, and CSS makes it look beautiful.

Why CSS is Important

  1. Separation of content and design – keeps HTML clean.
  2. Consistency – style multiple pages the same way.
  3. Responsive design – works for desktop, tablet, and mobile.
  4. User experience – makes websites attractive and readable.

CSS Syntax

CSS is made up of rules that tell the browser how to style HTML elements. Each rule consists of a selector and a declaration block.

Example of CSS Syntax


/* This is a CSS rule */
selector {
    property: value; /* Declaration */
    property2: value2; /* Another Declaration */
}

Components Explained

  • Selector: Specifies which HTML elements the rule applies to (e.g., h1, .class, #id).
  • Declaration Block: Contains one or more declarations enclosed in curly braces { }.
  • Declaration: Consists of a property and a value, separated by a colon (:). Each declaration ends with a semicolon (;).

Example Explained


/* Style all h1 elements */
h1 {
    color: blue; /* Set text color to blue */
    font-size: 24px; /* Set font size to 24 pixels */
}

In this example, the selector is h1, and the declaration block contains two declarations: color: blue; and font-size: 24px;.

CSS Selectors

Selectors are used to target HTML elements in CSS. They determine which elements will be styled by a rule.

Types of Selectors

  • Element Selector: Targets all elements of a specific type (e.g., p, div, h1).
  • Class Selector: Targets elements with a specific class attribute (e.g., .my-class).
  • ID Selector: Targets an element with a specific ID attribute (e.g., #my-id).
  • Universal Selector: Targets all elements on the page (*).

Example of Selectors


/* Element Selector */
p {
    color: green;
}
/* Class Selector */
.my-class {
    font-weight: bold;
}
/* ID Selector */
#my-id {
    background-color: yellow;
}
/* Universal Selector */
* {
    margin: 0;
    padding: 0;
}

Universal Selector

In CSS, the * { } syntax is called the universal selector.

It selects all elements on the page, allowing you to apply styles globally.

Example Usage


                        /* Apply a box-sizing rule to all elements */
                        * {
                            box-sizing: border-box;
                        }

                        /* Set a default font and background for the entire page */
                        * {
                            font-family: Arial, sans-serif;
                            background-color: #f0f0f0;
                        }

When to Use

  • Resetting default styles across all elements.
  • Applying global styles like font or background.
  • Ensuring consistent box-sizing for layout purposes.

Note of Caution

Using the universal selector can impact performance if overused, as it applies styles to every element. Use it judiciously for global styles or resets.


                        /* Example of using the universal selector */
                        * {
                            margin: 0;
                            padding: 0;
                            box-sizing: border-box;
                        }

Explanation:

                        margin: 0; → removes default margins of all elements
                        padding: 0; → removes default paddings
                        box-sizing: border-box; → ensures width/height calculations include padding and border
                        This combination is commonly used as a CSS reset to make layouts consistent across browsers.

How to Add CSS to HTML

There are three main ways to add CSS to your HTML documents:

  1. Inline CSS: Adding styles directly to an HTML element using the style attribute. 
    2. 
<h1 style="color: blue; font-size: 24px;">Hello World</h1>
  2. Internal CSS: Placing CSS rules within a <style> tag in the <head> section of your HTML document.
    3. 
<head>
    <style>
        h1 {
            color: blue;
            font-size: 24px;
        }
    </style>
</head>
  3. External CSS: Linking to an external CSS file using the <link> tag in the <head> section. This is the most common and recommended method for larger projects. 
    4. 
<head>
    <link rel="stylesheet" href="styles.css">
</head>

Best Practices

  • Use external CSS for better organization and maintainability.
  • Avoid inline styles to keep HTML clean and separate from design.
  • Use internal CSS for small projects or single-page documents.

CSS Comments

Comments in CSS are used to explain the code, make it more readable, and to temporarily disable parts of the code during development.

How to Write Comments


/* This is a single-line comment */
/*
   This is a multi-line comment 
    that spans multiple lines
*/

Example Usage


h1 {
    color: blue; /* This comment explains that the text color is set to blue */
    font-size: 24px; /* This comment explains that the font size is set to 24 pixels */
}

Best Practices

  • Use comments to explain complex sections of your CSS code.
  • Avoid over-commenting; only comment when necessary to improve understanding.
  • Keep comments up-to-date with code changes to prevent confusion.

CSS Errors

CSS errors can occur due to syntax mistakes, incorrect property names, or invalid values. These errors can lead to unexpected behavior in how styles are applied to your web pages.

Common CSS Errors

  • Missing Semicolon: Forgetting to end a declaration with a semicolon can cause the entire rule to be ignored. 
    
h1 {
    color: blue /* Missing semicolon here */
    font-size: 24px;
}
  • Incorrect Property Name: Using a property name that doesn't exist will result in no effect. 
    
h1 {
    colr: blue; /* Incorrect property name */
    font-size: 24px;
}
  • Invalid Value: Assigning a value that is not valid for a property will be ignored. 
    
h1 {
    color: bluish; /* Invalid value */
    font-size: 24px;
}

Debugging CSS Errors

  • Use browser developer tools to inspect elements and view applied styles.
  • Check the console for any CSS-related error messages.
  • Validate your CSS using online validators to catch syntax errors.

What are CSS Colors?

CSS colors are used to set the color of text, backgrounds, borders, and other visual elements on a webpage. There are several ways to define colors in CSS:

  • Color Names: Predefined color names like red, blue, green, etc.
  • Hexadecimal Values: Using a six-digit code preceded by a hash symbol (e.g., #FF0000 for red).
  • RGB Values: Using red, green, and blue values (e.g., rgb(255, 0, 0) for red).
  • HSL Values: Using hue, saturation, and lightness (e.g., hsl(0, 100%, 50%) for red).

Example of CSS Color Usage


/* Using color names */
h1 {
    color: blue;
}
/* Using hexadecimal values */
p {
    color: #FF5733; /* A shade of orange */
}
/* Using RGB values */
div {
    background-color: rgb(0, 255, 0); /* Green background */
}
/* Using HSL values */
section {
    border-color: hsl(240, 100%, 50%); /* Blue border */
}

Choosing Colors

When selecting colors for your website, consider the following:

  • Contrast: Ensure sufficient contrast between text and background for readability.
  • Branding: Use colors that align with your brand identity.
  • Accessibility: Choose colors that are accessible to all users, including those with color blindness.

CSS Backgrounds

CSS allows you to style the background of HTML elements using various properties. You can set background colors, images, and control their positioning and behavior.

Common Background Properties

Property Description Example
background-color Sets the background color of an element background-color: lightblue;
background-image Sets a background image for an element background-image: url('image.jpg');
background-repeat Controls if/how the background image repeats background-repeat: no-repeat;
background-size Specifies the size of the background image background-size: cover;
background-position Sets the starting position of the background image background-position: center;

Example Usage


/* Setting a background color */
body {
    background-color: #f0f0f0;
}
/* Setting a background image */
header {
    background-image: url('header-bg.jpg');
    background-repeat: no-repeat;
    background-size: cover;
    background-position: center;
}

Shorthand Properties

You can also use the shorthand background property to set multiple background properties at once.


section {
    background: url('section-bg.jpg') no-repeat center/cover lightgray;
}

What are CSS Margins?

CSS margins control the space around an element. They are the empty space outside the border of an element.

CSS Margin Properties

Property Description Example
margin-top Sets the top margin of an element margin-top: 10px;
margin-right Sets the right margin of an element margin-right: 10px;
margin-bottom Sets the bottom margin of an element margin-bottom: 10px;
margin-left Sets the left margin of an element margin-left: 10px;

Example Usage


/* Adding margins to a paragraph */
p {
    margin-top: 20px;
    margin-right: 15px;
    margin-bottom: 20px;
    margin-left: 15px;
}

Shorthand Margin Property

The margin property is a shorthand for setting all four margins at once.


/* All four margins are set to 10px */
div {
    margin: 10px;
}

/* Top and bottom margins are 10px, left and right are 15px */
div {
    margin: 10px 15px;
}

/* Top margin is 10px, right and left margins are 15px, bottom margin is 20px */
div {
    margin: 10px 15px 20px;
}

/* Top margin is 10px, right margin is 15px, bottom margin is 20px, left margin is 5px */
div {
    margin: 10px 15px 20px 5px;
}

CSS Box Model

The CSS box model describes the layout of HTML elements. Each element is represented as a rectangular box with four main components:

  • Content - The actual content of the element (text, images, etc.)
  • Padding - Space between the content and the border
  • Border - The border around the padding and content
  • Margin - Space outside the border (between elements)

CSS Box Model Properties

Property Description Example
padding-top Sets padding on top side of an element padding-top: 10px;
padding-right Sets padding on right side of an element padding-right: 15px;
padding-bottom Sets padding on bottom side of an element padding-bottom: 20px;
padding-left Sets padding on left side of an element padding-left: 15px;
border-width Sets the thickness of the border border-width: 2px;
border-style Sets the style of the border (solid, dashed, dotted, etc.) border-style: solid;
border-color Sets the color of the border border-color: red;
margin-top Sets the top margin of an element margin-top: 10px;
margin-right Sets the right margin of an element margin-right: 10px;
margin-bottom Sets the bottom margin of an element margin-bottom: 10px;
margin-left Sets the left margin of an element margin-left: 10px;

Example Usage


/* Example of CSS Box Model */
div {
    padding: 15px; /* Space inside the box */
    border: 2px solid black; /* Border around the box */
    margin: 20px; /* Space outside the box */
}

What is CSS Outline?

CSS outline is a line that is drawn around an element, outside the border edge. Unlike border, outline does not take up space and does not affect the layout of the element. It is used to highlight elements for accessibility purposes or visual emphasis.

CSS Outline Properties

Property Description Example
outline-style Sets the style of the outline (solid, dashed, dotted, etc.) outline-style: solid;
outline-color Sets the color of the outline outline-color: blue;
outline-width Sets the width of the outline outline-width: 3px;

Example Usage


/* Adding an outline to a button */
button {
    outline-style: dashed;
    outline-color: red;
    outline-width: 2px;
}

Shorthand Outline Property

You can also use the shorthand outline property to set all outline properties at once.


div {
    outline: 2px solid green;
}

What is CSS Grid?

CSS Grid is a layout system in CSS that allows you to arrange elements in rows and columns. It’s perfect for creating complex, responsive designs without using floats or positioning hacks. Think of it like a table, but much more flexible and modern.

CSS Grid Properties

These are applied to the grid container (the parent element).

Property Description Example
display Defines the container as a grid display: grid;
grid-template-columns Defines the number and width of columns grid-template-columns: 200px 1fr 2fr;
grid-template-rows Defines the number and height of rows grid-template-rows: 100px auto;
grid-template-areas Assigns names to areas for easier placement grid-template-areas: "header header" "sidebar main";
gap / row-gap / column-gap Space between rows and columns gap: 10px;
justify-items Horizontal alignment of grid items in their cells justify-items: center;
align-items Vertical alignment of grid items in their cells align-items: start;
justify-content Horizontal alignment of whole grid inside container justify-content: space-between;
align-content Vertical alignment of whole grid inside container align-content: center;

CSS Grid Item Properties

These are applied to grid items (child elements inside the grid).

Property Description Example
grid-column-start / grid-column-end Which columns an item starts/ends grid-column: 1 / 3;
grid-row-start / grid-row-end Which rows an item starts/ends grid-row: 2 / 4;
grid-area Assign an item to a named area grid-area: header;
justify-self Horizontal alignment inside the cell justify-self: end;
align-self Vertical alignment inside the cell align-self: center;

Units You’ll Commonly Use

  • px → fixed pixels
  • % → percentage of container
  • fr → fraction of available space (flexible)
  • auto → size based on content

Example


                        /* Grid container */
                        body {
                        margin: 0;
                        display: grid;
                        grid-template-rows: 80px 1fr 60px; /* header, main, footer */
                        grid-template-areas:
                        "header"
                        "main"
                        "footer";
                        height: 100vh;
                        }
                        /* Semantic areas */
                        header {
                        grid-area: header;
                        }
                        
                        main {
                        grid-area: main;
                        }
                        
                        footer {
                        grid-area: footer;
                        }
